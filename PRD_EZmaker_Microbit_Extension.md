# PRD: 이지메이커 마이크로비트 쉴드 MakeCode 확장 블록

## 1. 개요 (Overview)
- **목적:** 마이크로비트(micro:bit)와 결합되는 '이지메이커 쉴드보드'에서 다양한 이지메이커 센서 및 액추에이터를 손쉽게 제어할 수 있는 MakeCode용 확장(Extension) 블록 개발.
- **대상 사용자:** 블록 코딩 리터러시 교육을 받는 초/중/고등학생, 코딩 강사 및 일반 메이커.

## 2. 하드웨어 사양 (Hardware Specifications)
- **연결 표준:** 4핀 커넥터 사용
    - 배열: `1: VCC`, `2: GND`, `3: IO1`, `4: IO2`
- **핀 사용 원칙:**
    - 일반 아날로그 센서 및 디지털 입출력 모듈: **4번 핀(IO2)** 단일 사용.
    - I2C, Serial 등 통신 기반 모듈: **3번(IO1), 4번(IO2)** 핀 병합 사용.
- **포트 맵핑 (총 9개 커넥터):**
    - LED 매트릭스 공유 포트 제외.
    - **아날로그 입력 전용 포트 (3개):** P0, P1, P2
    - **디지털 입출력 전용 포트 (4개):** P8, P12, P13, P16
    - **I2C 전용 포트 (1개):** P19(SCL), P20(SDA)
    - **Serial 밎 2-Wire 전용 포트 (1개):** P14(RX / CLK), P15(TX / DAT)

## 3. 소프트웨어 요구사항 (Software Requirements)
- **플랫폼:** Microsoft MakeCode for micro:bit
- **형태:** Github 저장소를 통해 배포 및 Import 가능한 Custom Extension 형태 (TypeScript 활용).
- **기본 UI/UX:**
    - 아날로그/디지털 핀을 사용하는 센서들은 블록 내에서 **포트(Pin)를 드롭다운으로 선택**할 수 있도록 구성.
    - **I2C(P19, P20) 및 Serial/2-Wire(P14, P15) 통신용 센서들은 블록에 별도 핀 설정 없이 내부적으로 고정할당**하여 사용성 극대화.
    - 네오픽셀(NeoPixel), 서보모터 등 MakeCode 기존 확장이 존재하는 모듈은, 사용 편의성을 위해 **독립적인 이지메이커 전용 블록으로 래핑(Wrapping)하여 통합 제공**.

## 4. 지원 모듈 및 통신 방식별 분류

### 4.1. 아날로그 입력 및 디지털 입출력 (P0, P1, P2, P8, P12, P13, P16)
> 블록 인터페이스: 사용자가 핀 드롭다운 메뉴에서 포트 선택
- **아날로그 (P0, P1, P2 권장):** DIY-A 센서, 가스센서 (MQ2), 공기압센서, 밝기센서 (CDS), 소리센서, 자기장센서 (SLSS49E), 전압센서
- **디지털 (P8, P12, P13, P16 권장):** 초음파센서 (CS100A, 1핀 Echo/Trig 통합 제어), 네오픽셀 LED, 피에조 부저, 서보 모터, 레이저 포인트, 인체감지센서 (PIR), 스위치
- **아날로그 & 디지털 겸용:** DIY-B 센서

### 4.2. 1-Wire 센서 (디지털 포트 P8, P12, P13, P16 사용)
> 블록 인터페이스: 사용자가 핀 드롭다운 메뉴에서 포트 선택
- 고온센서 (MAX31850)
- 수중접촉온도센서 (DS18B20)
- 온도습도센서 (DHT11)

### 4.3. I2C 센서 (P19, P20 고정 포트 연결)
> 블록 인터페이스: 핀 선택 없음 (자동 할당: P19 SCL, P20 SDA)
- pH센서
- 기압센서 (BMP280)
- 이산화탄소센서 (SCD40)
- 자이로센서 (ICM-20948)
- 전류센서 (INA219)
- LCD (1602, 2004)

### 4.4. Serial / 2-Wire 센서 (P14, P15 고정 포트 연결)
> 블록 인터페이스: 핀 선택 없음 (자동 할당)
- 무게센서 (HX711): **14번 핀(Clock), 15번 핀(Data)** 고정 배치
- 미세먼지센서 (PMS7003): **14번 핀(RX), 15번 핀(TX)** 고정 배치

## 5. 단계별 개발 계획 (Implementation Phases)
- **Phase 1: 프로젝트 기초 뼈대 구성 및 기본 모듈 개발**
    - MakeCode 프로젝트 초기화 설정 (`pxt.json`, `custom.ts`)
    - 아날로그/디지털 입출력 모듈 통용 인터페이스 및 핀(P0~P16) Enum 드롭다운 정의
    - 기존 라이브러리(NeoPixel, Servo 등)를 래핑하여 이지메이커 전용 블록화 코어 작성
- **Phase 2: 단일 핀 복합 시그널 제어 및 1-Wire 모듈 적용**
    - 초음파거리센서(1핀 제어 모드) 동작 구현
    - 1-Wire 통신 프로토콜 (DHT11, DS18B20 등) 데이터 획득 블록 연동
- **Phase 3: 직렬 통신 프로토콜 통합 (고정 핀 사용)**
    - I2C 센서 레지스터 통신 세트 구현 (BMP280, SCD40 등) (P19, 20번 핀)
    - UART 시리얼 데이터 파싱 모듈 (PMS7003 등) 작성 (P14, 15번 핀)
    - 2-Wire 기반인 HX711의 클럭 발생 및 데이터 수신 블록화 구현 (P14, 15번 핀)
- **Phase 4: 에셋 매핑 및 외부 배포**
    - 각 블록별 로컬라이징(한/영 지원 등), 툴팁(Tooltip), 고유 아이콘 지정
    - Extension 활용 매뉴얼/README 정리 후 Github 배포 완성
